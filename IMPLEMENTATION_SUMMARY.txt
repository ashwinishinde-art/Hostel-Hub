❓ FAQ
═══════════════════════════════════════════════════════════════════════════════

Q: Will students be unregistered from the system?
A: NO! Students remain fully registered. Only room allocations change.

Q: Can I undo this operation?
A: YES! You can either:
   1. Manually re-allocate students through web interface
   2. Restore from database backup
   3. Use SQL to set status back to 'Active' if you know the IDs

Q: What data is lost?
A: NO DATA IS LOST! All preserved:
   ✓ Student accounts and logins
   ✓ Personal information
   ✓ Fee records and payment history
   ✓ Complaints and tickets
   ✓ Visitor requests
   ✓ All other system data

Q: Which method is fastest?
A: SQL is fastest, but web interface is most user-friendly.

Q: Which method is safest?
A: Web interface - includes visual confirmations.

Q: Can I do partial unallocation?
A: Not with these scripts. But you can:
   1. Use web interface to remove students one-by-one
   2. Use SQL with WHERE clause to target specific students

🔧 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

PROBLEM: "Can't connect to MySQL server"
SOLUTION: Start MySQL service
  $ sudo systemctl start mysql
  OR
  $ sudo service mysql start

PROBLEM: "Access denied for user"
SOLUTION: Check .env file for credentials
  • If password is empty, don't use -p flag
  • If password exists, use: mysql -u root -p

PROBLEM: "Database not found"
SOLUTION: Create database from schema
  $ mysql -u root < config/database.sql

PROBLEM: "ModuleNotFoundError: No module named 'MySQLdb'"
SOLUTION: Install dependencies
  $ pip install -r requirements.txt

For more troubleshooting:
→ See STUDENT_UNALLOCATION_COMPLETE.md → Troubleshooting section
