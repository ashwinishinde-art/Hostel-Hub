User → Login → "Forgot Password?" → Forgot Password Form
                                    ↓

                                    ↓

                                    ↓
User goes to verify OTP page
                                    ↓

                                    ↓
System validates OTP (format, expiry, match)
                                    ↓
Password Reset Form
                                    ↓
User enters new password (8+ characters)
                                    ↓
System validates and hashes password
                                    ↓
Password updated in database
                                    ↓
Confirmation email sent to user
                                    ↓
User redirected to login
                                    ↓
User logs in with new password ✅

📞 COMMON ISSUES & QUICK FIXES
═══════════════════════════════════════════════════════════════════════════════

"Email not sending"
  → Check .env has correct Gmail + app password
  → Run: python test_forgot_password.py
  → Check spam folder

"OTP expired"
  → Click "Resend OTP" button
  → Default expiry is 10 minutes
  → Configurable in .env

"Can't connect to database"
  → Start MySQL: sudo service mysql start
  → Initialize DB: mysql -u root -p < config/database.sql

"Wrong OTP message"
  → User can retry immediately
  → Or click "Resend OTP" for new code
